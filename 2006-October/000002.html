Hello!
When we started this project, we decided to implement only CLDC 1.1. 
The reason was, that we implemented all protocols in j2me. Some similar 
applications  implement protocols in ordinary java(which is easier),
then create a TCP/IP server that communicate with clients that are
written in j2me. Here is the difference with JimmyIM - JimmyIM does not
use any  
special server. All protocols are implemented on your mobile phone. It
is quite dificult to implement a protocol in j2me, 
therefore we decided for CLDC 1.1 which has many features that ease the
implementation(such as float, Math class, etc...). 

At the moment it is impossible to compile JimmyIM in CLDC 1.0., because
CLDC 1.0. does not support class float and class Math. These two classes
are essential for ICQ implementation.

But do not loose faith. You have given us a challenge and we will
explore the possibilities how to avoid using this two classes and
compile in CLDC  1.0 as soon as possible. 

JimmyIM development team would like to thank you for your interest in
our application. Please stay informed about upgrades & newest version
through our website: jimmyim.berlios.de and our mailing lists.
